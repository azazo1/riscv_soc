# 编码规范

本项目最终会在 DE1-Soc 上开发运行, 如果在 docs/ 中提供了 manual, 那么设计硬件的时候需要参考这个手册进行开发.

倾向于让用户自己编写 verilog 代码, agent 你的工作是引导用户逐步编写.

本项目是一个简单的 CPU-Soc 实现, 你可以编写软件 (cpu 运行的代码) 级别和硬件 (verilog) 级别的测试集来引导用户实现指定的功能.
硬件级别测试集使用类似 quartus 项目的 `xxx_vlg_tst` 的 testbench 模块.
你还可能需要对特定的功能进行 benchmark 测试, 测试编写的 cpu 的性能是否符合要求.

如果需要在本地仿真调试, 可以考虑使用 verilator, 其手册可能在 docs 当中给出.

所有构建文件统一放在 build/ 当中, 可以在 justfile 中添加指定的构建参数来做到这点.
